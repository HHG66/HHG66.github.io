基础

命令行工具

命令行工具本身不重要但是很多命令是很重要的。

dir：列出当前目录下所有文件。

cd：切换用户当前工作目录

环境变量： 操作系统中用来指定操作系统运行环境的一些参数，或者是提供给需要操作设备而不同设备路径不同，预约的一种操作环境机制。例如我们想在某个路径的cmd中打开不在这个文件目录的文件，正常是打不开的，配置文件目录到环境变量就可以正常打开了。

进程和线程

进程：程序运行的必备环境。

线程：计算机中最小计算单位，线程负责进程中程序。

单线程、多线程

Js是单线程

Nodejs是一个能够在服务器端运行js的开放源代码，跨平台js运行环境。

ES标准的缺陷

ES没有模块化系统，但是有模块化。

标准库比较少

缺乏管理系统

为了解决上述的缺陷，提出了CommonJs规范。

这样在node里面不在使用es的方式引入模块，而是使用require函数来执行,一般我们把这种的方式称为commonjs模块。require引入会返回一个对象，这个对象存储着模块的变量和方法等。

模块

node中分两种模块，一种是核心模块，一种是文件模块，核心模块是node本身提供的，文件模块自己自定义的。

node中存在global对象。

模块是一个函数，所以模块内所有的参数都是局部变量，除非我们指定为全局变量，模块包在一个如下的函数中

exports和module.exports本质是一样的，区别：exports是module.exports的一个引用，这就意味着我们直接修改exports会把引用修改掉，这就会导致无法访问到模块之前的 。

exports通过点的形式来导出，modules.exports既可以通过点也可以通过直接赋值形式导出。

module.exports公开了它指向的对象。 export公开了它指向的对象的属性。

包

凑合的包，记录需要的各种依赖关系，npm包是commonjs的一种实践。

全局包的安装位置查看 npm root -g

package-lock.json

已经了解了package.json的作用，引用package-lock.json是不是是一种多余的，不然，package.jsont提供了一种特殊的版本记录，例如〜0.13.0，代表只更新补丁版本，即0.13.1可以，0.14是不可以的，虽然补丁版本的更新一般不做大量的更改，但是还是有可能导致版本不一致导致的问题，所以我们需要记录更加明确的版本记录，以保证在其他地方可以正常使用，pack-lock.json就诞生， 他是在版本5以后引入的。所以在Git中需要将这个文件提交。当运行 npm update 时，package-lock.json 文件中的依赖的版本会被更新。

查看所有已安装的 npm 软件包（包括它们的依赖包）的最新版本，npm list

npm list -g 也一样，但适用于全局安装的软件包。

若要仅获取顶层的软件包（基本上就是告诉 npm 要安装并在 package.json 中列出的软件包），则运行 npm list --depth=0

可以使用@来安装指定版本：npm install <package>@<version>

列出软件包所有的以前的版本。 可以使用 npm view <package> versions

一般得，当发布新的版本时，开发者不仅仅是随心所欲地增加数字，还要遵循以下规则：

  • 当进行不兼容的 API 更改时，则升级主版本。
  • 当以向后兼容的方式添加功能时，则升级次版本。
  • 当进行向后兼容的缺陷修复时，则升级补丁版本。
