jQuery属性方法

属性和属性节点：首先了解区分属性和属性节点的知识，html元素的属性是属性节点，它存储在dom对象下attributes属性中。html元素对象才存在属性节点。更加通俗一点的讲，属性节点是html元素的属性，属性指的是js对象的属性。

属性相关方法

  • attr(name|pro|key,val|fn)：设置或返回被选元素的属性值，同时attr无法获取dom对象下的属性，只能获取属性节点。

注意attr方法只返回第一个对象的属性值。

  • removeAttr(name)：从每一个匹配的元素中删除一个属性
  • prop(n|p|k,v|f)：获取在匹配的元素集中的第一个元素的属性值。

prop可以获取dom元素对象的属性，而属性节点包含在dom元素对象中，所以也可以操作属性节点。官方推介在操作属性节点的时候返回值为true或者flase使用prop方法，其他情况使用attr方法。

  • removeProp(name)

attr可以获取html元素中的自定义属性，而prop无法获取自定义属性，可以获取元素本身存在的属性节点，例如class等。

类相关的方法

  • addClass(class|fn)：为每个匹配的元素添加指定的类名。
  • [removeClass(class|fn])：从所有匹配的元素中删除全部或者指定的类。
  • [toggleClass(class|fn,sw])：如果存在（不存在）就删除（添加）一个类。

文本值相关方法

  • [html(val|fn])：取得第一个匹配元素的html内容。这个函数不能用于XML文档。但可以用于XHTML文档。在一个 HTML 文档中, 我们可以使用 .html() 方法来获取任意一个元素的内容。 如果选择器匹配多于一个的元素，那么只有第一个匹配元素的 HTML 内容会被获取。
  • [text(val|fn])：取得所有匹配元素的内容。结果是由所有匹配元素包含的文本内容组合起来的文本。这个方法对HTML和XML文档都有效。
  • [val(val|fn|arr])：获得匹配元素的当前值。在 jQuery 1.2 中,可以返回任意元素的值了。包括select。如果多选，将返回一个数组，其包含所选的值

区别show和model(‘show’)

在bootstrap 中model(‘show’)方法是操作模态框的显示和关闭，出现隐藏的方法，而show()方法在jquery中是控制隐藏和显示，如果在模态框中直接使用的这个方法会使得模态框组件的方法失效；

jQuery操作数组

each静态方法

原生的foreach方法只能遍历数组，不能遍历为数组，jQuery可以遍历的数组和伪数组。

map静态方法

同样地原生map方法不能遍历静态方法。

each和map方法的区别：1、map方法默认返回一个空数组，each默认返回遍历的数组。

2、each不支持在回调函数中操作数组，map方法可以在return中对数组进行操作。

jQuery事件委托

动态添加的事件无法执行，那么就可以通过事件委托来解决，简单的说就是将事件监听对象指向父元素，这样只要保证父元素在动态添加的之前就存在，之后动态添加的子元素身上的事件都可以执行。
