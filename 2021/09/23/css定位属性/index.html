css定位属性

position属性指定元素定位类型，就是指定元素的在游览器的位置。

position属性有5个值，分别是：static、relative、fixed、absolute、sticky

​ 如果我们为元素指定了position属性之后，那么这个元素就有了top、bottom、left、right四个属性。这四个属性是为元素指定移动位置的，但是，position属性的不同，移动的结果是不同的！现在就讨论这五个的属性值的使用以及移动的结果。

static

这个值是position默认值，即没有定位，遵循正常的文档流。自然得就没有top、bottom、left、right四个属性。

relative

relative 相对定位；元素是指定了这个值之后，元素就具有得top、bottom、left、right四个属性，如果我们不指定这四个属性值，元素得位置会遵循文档流，在页面中是没有变化的，当我们指定了四个移动属性，那么元素的位置会相对于没变化的位置发现变化，这就是相对定位的最基本用法。

absolute

absolute 绝对定位；同样，元素指定了这个值之后，就具有得top、bottom、left、right四个属性，不指定这四个值，位置与原来文档流位置相同。元素会发生如下变化：

第一：元素脱离文档流。

第二：去寻找元素移动自身的位置。

元素会去一直往上寻找父元素是不是设定了position属性，并且值不为static、如果找到了这个元素，那么就设定absolute 的top、bottom、left、right四个初始值0的位置和这个元素上下左右边界位置相同。并且根据这个初始值去移动设定的值。

注意：是根据上下左右边界去初始化0的位置，例如寻找到的这个元素宽高是一个200px*200px的盒子，那么right位置就要在盒子右边界的基础去移动，例如：设定absolute父元素position值为relative 指定absolute的right值为0，

第二：元素脱离文档流，如果父元素没有指定position，那么absolute 会以html标签为初始值去移动位置。

fixed

元素的位置相对于浏览器窗口是固定位置。即使窗口是滚动的它也不会移动;Fixed定位使元素的位置与文档流无关，因此不占据空间,因此Fixed定位的元素和其他元素重叠。

sticky

粘性定位，基于用户的滚动位置来定位。

如果没有滚动条，那么行为就像是相对定位。
