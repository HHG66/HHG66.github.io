Vue学习

简介：

两大特性：

1、数据驱动视图

2、双向数据绑定

MVVM编程思想

双向数据绑定是一MVVM思想的实现，MVVM分别代表Model、View、ViewModel。这个思想是把每一个html页面拆分成三个部分，数据源、页面的DOM结构、vue的实例三个部分。

ViewModel是连接视图和模型层的中间部分，他要同时监听view层和Model改变，来相对应的改变另一个的值。

挂载点

el：挂载点

作用：相当于原生js中的选择器。

作用范围：在el命中的内部。

支持的选择器：id选择器、类选择器。注意挂载点不能命中html、body标签，会报错。

data数据属性

可以写简单数据类型、复杂类型。

vue指令

按照用途可以分类：

  1. 内容渲染指令

    • 插值表达式

    • v-text

    • v-html

  2. 属性绑定指令

    • v-bind

  3. 事件绑定指令

    • v-on

  4. 双向绑定指令

    • v-model

  5. 条件渲染指令

    • v-if
    • v-show

  6. 列表渲染指令

    • v-for

插值表达式

{{ msg }} 把msg替换成设置的值。内容只会替换msg

v-text

标签文本值。覆盖原有内容。

v-html

标签的innerHTML

v-bind

设置元素的属性。使用在元素的属性，不能用在元素本身。

v-bind:属性名=属性值。简写 ：属性名=属性值

v-on

元素绑定事件

支持两种写法：1、v-on：事件=”事件名称”

2、@事件 =”事件名称”

原生事件对象：$event

v-model

双向数据绑定指令，视图和数据绑定在一起。只能和表单元素使用；如下几种

  • <input>
  • <select>
  • <textarea>
  • components

修饰符

  • .lazy
  • .number
  • .trim

v-if

根据表达式的真假，切换元素的显示和隐藏（操作的是dom元素），本质是在dom中增加元素。

v-show

根据表达式真假确定元素的隐藏和显示。

v-for

根据数据生成列表结构，注意使用的时候需要绑定一个属性key，尽量使用唯一的id去绑定，而不是使用索引，索引是没有意义的。

事件修饰符

修改默认事件的触发机制。阻止事件冒泡等情况或者键盘事件，按下特性的按钮触发事件。

  • .stop
  • .prevent
  • .capture
  • .self
  • .once
  • .passive

过滤器

作用：文本格式化

使用：插值表达式、v-bind属性绑定。

在filters中定义一个处理函数，在vue 3不再支持过滤器。

计算属性和侦听器

计算属性

作用：插值表达式本意进行简单的逻辑js运算，当进行复杂的js逻辑运算会导致模板过重和难以维护，所以当你想要在插值表达式中进行复杂的运算时，要想到使用计算属性。

侦听器

监视数据变化，一般形况下计算属性更都使用，但是会有一些情况下会使用到怎挺起，vue提供了更加通用的监听数据办法的方法就是侦听器。

  1. 函数格式的侦听器
  • 缺点
    • ​ 无法进入页面的时候自动执行一次，通过对象格式侦听器解决。
  1. 对象格式的侦听器
  • 缺点
    • 进行页面自动执行一次侦听，使用 immediate选项指定。
    • 侦听一个对象，对象内部属性变化，不会触发侦听，使用deep选项解决。

如果监听对象的子属性，需要在监听名称上增加一层单引号

axios

一个网络请求库

axios挂载到Vue原型上配置根路径。

优点：使用方便，封装度高

缺点：无法实现api接口的复用。

单页面应用程序

一个web网站只有一个的html页面，所有功能都在一个html页面完成。

项目结构：src是源代码，assets项目中静态资源文件，main.js是项目的入口文件，public是静态文件其他的是配置文件和依赖文件夹。

样式冲突

组件的样式一般全局生效，会造成样式冲突，解决办法1通过增加自定属性，这样增加的样式都是指定在元素中有自定义属性的元素，第二种方法是在style标签中增加一个scoped，效果和增加自定义属性一致。

/deep/的用法：能够以穿透组件样式，影响它的子组件样式。

vue组件的实例对象

游览器只能认识html格式的文件，vue结尾的文件游览器不识别，中间是有一个编译的过程，其中依赖vue-template-compiler包去将vue文件编译成js加载到html中。

每一个组件都是一个vue的实例对象，在使用这个组件的使用是new了这个实例调用。

生命周期

一个组件创建-运行-销毁的整个过程称为生命周期，强调一个时间段。

因为是单页面应用，所以组件就在一个页面中，这样组件就需要销毁。当想在某个组件创建前等状态下执行某些事件，vue也提供了这种的函数，这种函数就是生命周期函数，它强调的是事件点。

创建阶段方法：

​ 创建前：beforCreate

​ data、methods等都未创建的时间点。

​ 创建完成：created

​ 渲染前：beforMount

​ 渲染时：mounted

组件运行阶段的方法：

​ beforeUpdte

​ update

组件销毁阶段:

​ beforeDestroy

​ destroy

生命周期函数不推介使用箭头函数，因为箭头函数不存在this，会导致undefined的情况。

ref引用

jquery库的优势是简化了操作DOM的过程，vue的优势就是MVVM设计模型，使得不需要复杂的操作DOM，设计人员只需要把数据维护好就行。

实际中还是不可避免的的会操作DOM，就是ref引用。

在目标元素增加一个属性ref，注意值不能和其他的ref值相同， 相同后面的值会把前面的值覆盖。

两个组件中，一个组件的数据发生变化，另一个组件使用了ref的引用会报错，因为数据发生变化，dom结构还没有渲染成功，可以使用vue提供的$.nextTick(cd)方法，cd是回调函数，执行代码放在回调函数中。

使用整个而不是updata生命周期函数中，是因为在updata函数中，数据每次发生变化都会触发的函数，只需要触发一次的需要就不能满足。

动态组件

动态切换组件的隐藏和显示。

<component>是vue的内置组件标签。

组件切换是重新创建和销毁的，想要保持之前没有销毁的状态可以使用keep-alive保持之前的状态，这时候组件隐藏是未激活的状态，显示的是激活状态。

监听组件激活的状态:

当使用keep-alive后想要监听组件的激活状态那么vue提供的了生命周期函数。

组件缓存的时候，会触发组件的deactivated生命周期函数，组件激活的时候会触发组件的activated生命周期函数。

keep-alive的include属性，指定组件是否被缓存，以及对应的exclude，注意两者不能同时使用。

组件注册名称和组件声明名称的区别：

声明组件的时候没有指定name名称，组件默认名称为声明名称。

注册名称的作用时在页面结构以标签形式使用，而声明名称主要作用是，调试工具中看到组件名称和keep-alive实现缓冲。

插槽

提供组件封装的能力，运行开发者在封装组件时，把不确认的、希望用户指定的部分定义为插槽。

<slot>标签提供一个占位符，允许将不同的标签填充到这个标签中，我们把整个标签称为插槽。

v-slot指令会将指令的值填充到插槽的name属性中，插槽name属性默认时``default`。

v-slot指定只能在template标签或者组件中使用。

template标签是一个包裹性的区域，不会渲染成任何一个元素。

v-slot指令有简写形式：#

slot标签内容区域可以添加内容，当没有指定插槽内容时候，可以默认显示slot标签区域的内容，标签内的内容称为后备内容。
